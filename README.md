# 🛋️ معرض عباد الله للأثاث — نظام الإدارة السحابي (نسخة Firebase)

نظام إدارة كامل (منتجات، عملاء، فواتير، تقسيط، موردين، موظفين، مصروفات) مبني بـ React،
يخزن البيانات على **Firebase Firestore** (قاعدة بيانات سحابية مجانية من جوجل) بحيث يفتح
النظام من أي جهاز عبر لينك واحد، وأي تعديل يتزامن فوراً مع كل الأجهزة.

**التكلفة: 0 جنيه** (Firebase + GitHub + Vercel كلها مجانية للاستخدام الصغير/الشخصي).

> ملاحظة: النسخة دي كانت أصلاً على Supabase، وتم تحويلها بالكامل لتستخدم Firebase
> بسبب عطل مؤقت كان حاصل في Supabase. الوظائف والشاشات كلها نفسها بدون أي فرق.

---

## الخطوات بالترتيب (نفّذها مرة واحدة فقط)

### الخطوة 1: إنشاء مشروع Firebase وقاعدة البيانات

1. افتح https://console.firebase.google.com وسجّل دخول بحساب جوجل.
2. اضغط **إضافة مشروع / Add project**، سمّه مثلاً `ebadallah-erp`، وكمّل الخطوات
   (تقدر تعطّل Google Analytics، مش لازم للنظام) حتى ينشئ المشروع.
3. من الصفحة الرئيسية للمشروع، اضغط أيقونة الويب **`</>`** لإضافة تطبيق ويب جديد.
   - اكتب اسم مستعار زي `ebadallah-web` واضغط **Register app**.
   - هيظهر لك كود فيه كائن اسمه `firebaseConfig` — **انسخه كله بالكامل** (هتحتاجه في الخطوة 4)، شكله كده تقريباً:
     ```js
     const firebaseConfig = {
       apiKey: "AIza...",
       authDomain: "ebadallah-erp.firebaseapp.com",
       projectId: "ebadallah-erp",
       storageBucket: "ebadallah-erp.appspot.com",
       messagingSenderId: "123456789",
       appId: "1:123456789:web:abcdef"
     };
     ```
   - اضغط **Continue to console**.
4. من القائمة الجانبية: **Build → Firestore Database → Create database**.
   - اختر **Start in production mode**.
   - اختر أقرب موقع سيرفر (مثلاً `eur3 (europe-west)`)، واضغط **Enable**.
5. بعد إنشاء القاعدة، روح تبويب **Rules** بالأعلى، امسح المحتوى الموجود، والصق بدلاً
   منه محتوى ملف `firestore.rules` الموجود في هذا المجلد، ثم اضغط **Publish**.
6. لازم تعمل أول مستخدم (admin) يدوياً مرة واحدة:
   - روح تبويب **Data**، اضغط **Start collection**.
   - Collection ID اكتب: `users` ثم Next.
   - في **Document ID** اختَر **إدخال يدوي** واكتب الرقم `1` (بدل Auto-ID).
   - أضف الحقول دي بالظبط (بالنوع الصحيح لكل حقل):

     | الحقل      | النوع    | القيمة              |
     |------------|----------|---------------------|
     | name       | string   | مدير النظام          |
     | username   | string   | admin               |
     | password   | string   | admin123            |
     | role       | string   | مدير النظام          |
     | active     | boolean  | true                |

   - اضغط **Save**.

### الخطوة 2: رفع المشروع على GitHub

1. افتح https://github.com واعمل حساب لو مالكش واحد.
2. اضغط **New repository**، سمّيه `ebadallah-erp`، ثم **Create repository**.
3. ارفع كل ملفات هذا المجلد للـ Repository (بما فيها مجلد `src` بالكامل):
   - في صفحة الـ Repository اضغط **uploading an existing file**.
   - اسحب كل الملفات والمجلدات وارفعها.
   - اكتب أي رسالة commit واضغط **Commit changes**.

   (لو بتستخدم Git من التيرمينال بدلاً من كده):
   ```bash
   git init
   git add .
   git commit -m "أول رفع للمشروع"
   git branch -M main
   git remote add origin https://github.com/USERNAME/ebadallah-erp.git
   git push -u origin main
   ```

### الخطوة 3: الاستضافة المجانية على Vercel

1. افتح https://vercel.com وسجّل دخول بحساب **GitHub** مباشرة.
2. اضغط **Add New... → Project**.
3. هيظهر لك الـ Repository اللي رفعته (`ebadallah-erp`) — اضغط **Import** بجانبه.
4. Vercel هيكتشف تلقائياً إنه مشروع Vite/React، سيبه على الإعدادات الافتراضية.
5. اضغط **Deploy** وانتظر دقيقة أو اتنين (هيعمل `npm install` وهيثبّت مكتبة Firebase تلقائياً).
6. بعد ما يخلص هتلاقي لينك زي: `https://ebadallah-erp.vercel.app`
   — **ده اللينك اللي هيفتح بيه النظام من أي جهاز أو موبايل في العالم.**

### الخطوة 4: أول تشغيل وربط قاعدة البيانات

1. افتح اللينك اللي طلع من Vercel على أي متصفح.
2. هتظهر لك صفحة "إعداد" فيها مربع نص كبير — الصق فيه كائن `firebaseConfig`
   اللي نسخته في الخطوة 1 **كامل زي ما هو** (حتى لو فيه `const firebaseConfig = ... ;`
   حواليه، مفيش مشكلة، النظام بيتعرف عليه لوحده).
3. اضغط **حفظ وتشغيل**. الإعدادات دي بتتسجل في متصفحك فقط (localStorage)، فلازم تعمل
   نفس الخطوة دي مرة واحدة بس في **كل جهاز/متصفح جديد** هتفتح منه النظام.
4. بعدها هتظهر شاشة تسجيل الدخول:
   - اسم المستخدم: `admin`
   - كلمة المرور: `admin123`
   - **مهم:** غيّر الباسورد ده بعد أول دخول من Firebase Console مباشرة
     (Firestore Database → Data → users → المستند رقم 1 → عدّل حقل password).

## هل النظام هيعمل سينك تلقائي فعلاً؟

نعم. كل عملية (إضافة منتج، فاتورة، مصروف...) بتتبعت فوراً لقاعدة بيانات Firebase السحابية.
لو فتحت النظام من جهاز تاني (موبايل، لابتوب، فرع تاني) بنفس اللينك (وعملت خطوة الإعداد
مرة واحدة على الجهاز ده)، هتشوف كل البيانات لحظياً لأنها كلها متخزنة في نفس المكان في السحاب.

## تحديث النظام مستقبلاً

لو غيّرت أي حاجة في الكود بعد كده:
1. عدّل الملفات في GitHub (أو من جهازك وادفعها بـ `git push`).
2. Vercel هيكتشف التحديث تلقائياً وينشره خلال ثوانٍ — مفيش خطوات إضافية.

## ملاحظة أمان

الإعداد الحالي (`firestore.rules`) يسمح لأي حد معاه لينك النظام + بيانات الدخول بالوصول
للبيانات مباشرة (وده طبيعي للتطبيقات الداخلية الصغيرة). لو عايز حماية أقوى مستقبلاً
(صلاحيات مختلفة لكل مستخدم، تسجيل دخول حقيقي مشفّر) نقدر نطور الإعداد باستخدام
**Firebase Authentication** الحقيقي بدل التحقق اليدوي من مستند `users`.

    └── App.jsx           كل شاشات النظام (لوحة تحكم، منتجات، عملاء، فواتير...)
